# ADR 002: Stack Technique

## Statut
Accepté

## Contexte
Nous avons besoin d'une stack technique moderne, performante, typée et adaptée à un écosystème fullstack JavaScript/TypeScript pour développer le Parking Reservation System.

## Décision
La stack technique retenue est la suivante :
- **Langage** : TypeScript
- **Backend / Serveur** : Node.js avec le framework Fastify pour ses hautes performances.
- **Frontend** : React

## Conséquences
- **Avantages** : Code robuste et maintenable grâce au typage fort (TypeScript). Partage potentiel de types entre le back et le front. L'écosystème Node.js / React est riche et éprouvé
Maitrise de la stack technique par l'équipe
- **Inconvénients** : Gestion stricte des types requise
